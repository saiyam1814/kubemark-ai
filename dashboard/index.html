Understanding NCCL Benchmark Results

Bus Bandwidth (GB/s)

Measures the effective data transfer rate across all GPUs for collective operations. This is the key metric — it tells you how well your GPUs can work together. NVLink-connected GPUs (H100, A100 in DGX) achieve 200-300+ GB/s. PCIe-connected GPUs (L40S, RTX) typically reach 15-25 GB/s.

Algorithm Bandwidth (GB/s)

Raw data moved divided by time. Bus bandwidth adjusts for the communication pattern (e.g., AllReduce sends data across a ring), making it the fairer comparison metric across different GPU counts.

Message Size

The amount of data exchanged per operation. Small messages (KB) test latency. Large messages (MB-GB) test bandwidth. Real training workloads use large messages, so the right side of the chart matters most.

What's “good”?

Run the same benchmark multiple times — consistent results mean stable infrastructure. For multi-GPU runs, compare against published community results for your GPU type (see reference table below). Single-GPU runs measure memory bandwidth, not inter-GPU communication.

TFLOPS per GPU

Trillion floating-point operations per second per GPU. Measures how efficiently GPUs are utilized during training. Higher = better hardware utilization.

Understanding Training Benchmark Results

TFLOPS/GPU

Trillion floating-point operations per second per GPU. This is the primary metric for training throughput. It captures both compute and communication efficiency. Higher means your GPUs are spending more time computing and less time waiting.

Step Time (ms)

Time per training iteration. Lower is better. This directly translates to training speed: halving step time = training twice as fast. Affected by model size, parallelism strategy, and GPU interconnect speed.

Iterations 35-44

We measure iterations 35-44 (not the first ones) because early iterations include warmup effects like JIT compilation and memory allocation. This matches NVIDIA's official measurement methodology.

Parallelism

Training uses Tensor Parallel (TP), Pipeline Parallel (PP), Expert Parallel (EP), and Data Parallel (DP). The parallelism config is auto-selected based on your GPU type to match NVIDIA's optimal settings.
